# Validation Ladder

Use this checklist so Korean synthetic consumer panels do not get mistaken for proof.

## 1. Synthetic panel

- [ ] Persona source documented (`example JSON`, `HF sample`, or curated panel).
- [ ] At least one skeptical persona included.
- [ ] Output labels findings as `[hypothesis]`.
- [ ] Main objections and proof requirements extracted.

## 2. Real customer check

- [ ] Convert synthetic signals into 5-8 non-leading interview questions.
- [ ] Interview at least 3 target users or sales-proxy stakeholders.
- [ ] Mark repeated findings as `[experimental]`; keep one-off reactions as `[hypothesis]`.

## 3. Behavioral test

- [ ] Test revised copy/prototype with a landing page, sales DM, proposal intro, or demo.
- [ ] Capture actual behavior: reply, click, meeting booked, willingness to share data, or payment signal.

## 4. Decision memo

- [ ] Summarize: keep / change / remove.
- [ ] Separate synthetic, interview, and behavioral evidence.
- [ ] Decide GO / PIVOT / KILL with confidence tag.
